## 🛑 Executive Summary

`www.basquetcatala.cat` is a PHP 8.4 site that gates **every URL** (except `/robots.txt`) behind a custom Google reCAPTCHA v3 challenge wall at `/security-check` that issues an HMAC-signed `fcbq_rc` cookie after a successful POST to `/recaptcha/verifica`. UA spoofing (Googlebot, Twitterbot) does NOT bypass it, so `httpx` alone cannot fetch any data page.

Once past the wall, calendar + standings ARE server-rendered HTML at `/competicions/resultats/{numericId}/{jornada}` (parseable with `beautifulsoup4`), but **player box scores are NOT in those pages**:
- `/acta/<id>` returns an HTML shell + a presigned S3 JPG image of the paper scoresheet (~1h `Expires` TTL). No machine-readable stats.
- Real digital stats live in a React SPA at `/estadistiques/<mongoObjectId>` that hydrates from a third-party microservice (`msstats.optimalwayconsulting.com/v1`) using a short-lived (60s – 15min) RS256 JWT embedded in the HTML.

FCBQ's own sanctioned channel for programmatic data access is **OPEN DATA FCBQ**, a paid service (~€25-30/month + VAT) requiring federated-club status and IT coordination.

**Verdict:** the current LOCKED architectural constraints (`httpx`-only, zero new top-level deps, `beautifulsoup4` parsing, offline batch) are incompatible with the actual scraping surface for the data we want (player box scores). Proceeding without re-scope will produce a parser that crashes on its first request.

### 2. `comp_url_pattern`
- **Real pattern:** `/competicions/resultats/{comp_id:int}` and `/competicions/resultats/{comp_id:int}/{jornada:int}`
- `{comp_id}` = **numeric internal FCBQ ID**, NOT a human slug. The slug `cc-2a-m` in our planning docs is Roger's internal naming, not an FCBQ URL token.
- `{jornada}` 1..N = specific round; `0` = all rounds; omit for default ("last played round")
- **Verified examples** (Wayback + WebSearch):
  - `/competicions/resultats/19773` → *lliga catalana tercera feb - fase grups - 06* (2025-26)
  - `/competicions/resultats/19773/3` → Jornada 3 (21-09-2025)
  - `/competicions/resultats/19801` → *liga acb - lliga catalana acb - fase grups - 01* (2025-26)
  - `/competicions/resultats/17083`, `/17084`, `/17085` → Copa Catalunya / CC primera categoria 2024-25
  - Legacy alphanumeric form: `/competicions/resultats/03511001-1` (pre-migration ID format)
- **Discovery of `comp_id` for 2a Catalana Masculina 2025-26 requires either:**
  - (a) Search form at `/cerca_competicions/resultats` — behind the reCAPTCHA wall
  - (b) Season list at `/llistatCompeticions/temporades` — also behind the wall
- **JS taxonomy hint** (from comp results page): `carregaCategories('M',1); carregaCom('M', 1, 1160);` — params likely `(gender, category_tier, comp_family_id)`. `M`=Masculí, `1`=top tier, `1160`=internal family ID.
- **Confidence:** HIGH on URL template. MEDIUM-LOW on deriving the specific `comp_id` for `cc-2a-m 2025-26` without manual interaction with the search form.

### 3. `game_url_pattern`
**Two parallel URL families per game — they serve different content:**

| Family | URL template | Content type | Useful for |

| Acta (paper scoresheet) | `/acta/{int_id}` e.g. `/acta/342` | HTML shell + presigned S3 JPG image of the paper acta + match metadata (referees, venue, jersey colors, date/time) | Game metadata only; **NOT** machine-readable box scores |

### 7. `session_required`
**YES.** Three layers:

| Cookie / Token | Set by | Purpose | TTL |
|---|---|---|---|
| `fcbq_rc` | `/security-check` redirect | reCAPTCHA verification state (JSON payload + HMAC) | Set on first hit (`pending`), flips to `verified` after `/recaptcha/verifica` succeeds |
| `PHPSESSID` (implicit) | PHP framework | Standard PHP session | Browser session |
| `data-jwt` (embedded in HTML, not a cookie) | Server HTML on `/estadistiques/*` pages | Authorizes calls to `msstats.optimalwayconsulting.com` from React SPA | Very short (60s – 15min observed in Wayback samples) |

- `fcbq_rc` payload (base64 JSON): `{"status":"pending","verified_until":0,"score":null,"action":null,"updated_at":<unix>}.<hmac_hex>`. After successful `/recaptcha/verifica`, `status` → `verified` and `verified_until` → future timestamp.
- reCAPTCHA site key (publicly visible in `/security-check` HTML): `<recaptcha-sitekey-public-redacted-for-secret-scanner>`.

### 9. `gotchas` (priority order)

1. **🚨 reCAPTCHA v3 wall on EVERY URL** — global, server-enforced, no UA bypass. Site key `<recaptcha-sitekey-public-redacted-for-secret-scanner>`. **This alone breaks D2.5-02 / D2.5-09.**
2. **🚨 Acta = JPG image, not HTML** — `/acta/<id>` extraction is impossible without OCR on a printed scoresheet (poor quality). Real stats are at `/estadistiques/<id>`.
3. **🚨 Stats SPA needs JWT** — `data-jwt` expires in 60s–15min (RS256). Stats live on third-party `msstats.optimalwayconsulting.com/v1`. Token reuse window is narrow.
4. **Slug ≠ FCBQ ID** — `cc-2a-m` is our internal naming. FCBQ uses integer IDs (`19773`, `19801`) and MongoDB ObjectIDs (`68ceefbb84f669000119374a`). Discovery step required to map "2a Catalana Masculina 2025-26" → `comp_id`.
5. **No standalone standings URL** — must request the latest-jornada calendar page to get standings (same page). Use `/competicions/resultats/{id}/0` for all-jornadas-flat view (useful for backfill).
6. **Pagination via jornada selector only** — no `?page=` query param; iterate `jornada=1..N_total` discovered from `<select>` options.
7. **Acta JPG URLs are AWS-presigned with ~1h `Expires`** — must download eagerly or refetch the page after expiry.
8. **Two ID systems for the same game** (`/acta/<int>` and `/estadistiques/<mongoId>`) — no documented mapping; scrape both from the calendar page.
9. **No anti-bot signals beyond reCAPTCHA** — once past the wall, no rate limiting evident from headers. Sustained scraping after captcha solve is untested in this spike.
10. **OPEN DATA FCBQ is the official paid path** (~€25-30/mo + VAT, requires federated-club status + IT coordination).
11. **Avís Legal exists** at `/avis-legal` (behind wall — not directly fetched). No explicit anti-scraping clause confirmed, but reCAPTCHA wall is a strong implicit signal.
12. **No public sitemap.xml** — `/sitemap.xml` also redirects to `/security-check`.
