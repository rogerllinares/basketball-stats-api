## Interview-prep cheat-sheet (Roger to review later)

When asked "Why did you build the ingest as offline batch instead of live fetch?":
> "Three reasons. Production runtime stays pure — no external HTTP, so API uptime doesn't depend on FCBQ's. Fixtures are versioned in git, fully diff-able and replayable, which is what you want for a deterministic test environment. And no API keys or secrets needed for prod deploys. The trade-off is data goes stale until next manual scrape, which is acceptable for a portfolio API — recruiters aren't expecting real-time scores. The decision is documented in ADR-0005 with the four alternatives I rejected."

When asked "Why two-phase fetch + parse?":
> "ETL separation of concerns. Fetch handles HTTP — rate limiting, retries, UA rotation. Parse handles transformation — raw HTML to validated JSON. Splitting them means: I can re-run parse a hundred times without re-hitting FCBQ; if the parser breaks because FCBQ changes a CSS class, I debug against cached raw; and the scrape is resumable — if it dies at game 47 of 120, the next fetch picks up from there."

When asked "Why UPSERT instead of DELETE+INSERT?":
> "Idempotent re-runs and FK preservation. UPSERT keyed on natural keys (license_id for players, fcbq_game_id for games) means running the loader twice is a no-op. DELETE+INSERT would cascade and wipe any auth-related coach metadata added by Phase 3 — UPSERT only touches the rows the fixture knows about."

When asked "Why argparse instead of typer/click?":
> "Stdlib is enough. Three sub-commands — fetch, parse, scrape — fit in 30 lines of argparse sub-parsers. Adding typer is +1 transitive dep for marginal ergonomic gain. The signal is: reach for stdlib when it covers the surface, reach for libraries when stdlib doesn't. Recruiters reading dependency lists notice the discipline."

When asked "How did you enforce no-live-ingest at the code level?":
> "CI step grepping the import graph: `grep -r 'from basketball_stats.ingest' src/basketball_stats/ --exclude-dir=ingest` and failing the build on any match. Doc-level promises rot; CI gates don't. Plus the loader lives outside the package (`data/seed/load_fcbq.py`) so production FastAPI code has no reason to import it."
