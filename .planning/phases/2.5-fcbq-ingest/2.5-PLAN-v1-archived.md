## Wave 3 — Loader (the data path — BEFORE any scraping)

**Rationale for ordering:** Build the loader first against the handcrafted sample. This proves the data→DB path independently of the scraper. If the loader works on `sample-mini.json`, it works on any valid `FCBQFixture`-shaped JSON. Then we know any later parser bugs are PARSER bugs, not loader bugs. Production-grade discipline: isolate failure surfaces.

### Task 8 — Write load_fcbq.py

**Issue:** `enhancement` + `P1` + `phase/2.5-fcbq-ingest` — "Write data/seed/load_fcbq.py UPSERT loader (D2.5-06)"
**Branch:** `feat/<N>-load-fcbq-loader`

**File:** `data/seed/load_fcbq.py`
**Pattern:** Mirror `src/basketball_stats/seed/minimal.py` structure — argparse `--force` flag, `get_session_factory()` bootstrap, idempotent. **Critical difference:** uses `sqlalchemy.dialects.postgresql.insert(...).on_conflict_do_update(...)` per D2.5-06.

**Logic:**
1. Argparse: positional `fixture_path`, optional `--dry-run`, `--verbose`.
2. Read JSON, parse with `FCBQFixture` (raises on schema mismatch).
3. Single transaction:
   - UPSERT competition (`ON CONFLICT (slug, season)`)
   - UPSERT clubs (derived from teams, `ON CONFLICT (slug)`)
   - UPSERT teams (`ON CONFLICT (fcbq_team_id)`)
   - UPSERT players (`ON CONFLICT (license_id)`)
   - UPSERT rosters (M:N player-team per season, `ON CONFLICT (player_id, team_id, season)`)
   - UPSERT games (`ON CONFLICT (fcbq_game_id)`)
   - UPSERT box_scores (`ON CONFLICT (game_id, player_id)`)
4. Print summary: rows inserted vs rows updated vs rows unchanged.

**Crucial detail:** Loader writes ONLY the inputs to GENERATED columns (`val`, `reb`). Postgres computes them on INSERT/UPDATE.

**Verification:**
```bash

# Migration round-trip still passes
```

**Surface:**
- `class FCBQClient(base_url, rate_limit_seconds=1.0, jitter=(0.0, 0.5), user_agents=DEFAULT_UA_POOL)`
- `async def fetch(self, path: str) -> bytes` — returns raw response body
- Internal: `tenacity.AsyncRetrying(stop=stop_after_attempt(5), wait=wait_exponential(multiplier=2, min=2, max=30), retry=retry_if_exception_type(httpx.HTTPError) | retry_if_result(lambda r: r.status_code >= 500))`

## Decision Tree (referenced from individual tasks)

This section anchors edge cases — if you hit one of these during execution, follow the rule, don't re-derive.

| Situation | Rule | Source |
|---|---|---|
| FCBQ page returns 403/anti-bot | Retry with different UA from pool; if 3 consecutive failures, log + skip resource + continue. STOP entire scrape if 50% resources fail. | D2.5-08 |
| Player has no published license_id | SKIP the player + emit structured warning to stderr. Do NOT invent IDs. | D2.5-05 |
| FCBQ HTML structure changed mid-scrape | Parser raises `FCBQParseError` with raw_path; CLI exits non-zero. Operator inspects raw, updates parser. | D2.5-08 |
| Loader hits FK violation (e.g., box_score references missing player) | Fail the entire transaction — fixture is invalid. Roger reviews. Do NOT partial-load. | D2.5-06 |
| Wave 0 spike says Playwright needed | STOP. Re-discuss with Roger — Playwright violates D2.5-02 + D2.5-09 (zero new deps). | D2.5-01 |
| Multiple competitions hit at once during one scrape | NOT THIS PHASE — Wave 5 ships only cc-2a-m 2025-26. Other comps are follow-up PRs. | D2.5-07 |
| CI import-ban grep matches a legit case | There's no legit case — that's the whole point. If it matches, the import is wrong. Don't add an exclusion. | D2.5-10 |
